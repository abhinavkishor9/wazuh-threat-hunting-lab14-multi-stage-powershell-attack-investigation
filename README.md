# wazuh-threat-hunting-lab14-multi-stage-powershell-attack-investigation
## Overview

This lab demonstrates how to investigate a multi-stage attack by correlating multiple Sysmon events within Wazuh SIEM.

Instead of analyzing a single event, this investigation reconstructs an attack timeline by linking Process Creation, Network Connection, and File Creation events generated from PowerShell activity.

This approach closely resembles how Security Operations Center (SOC) analysts investigate endpoint incidents in enterprise environments.

---

# Lab Objectives

- Generate PowerShell activity
- Generate outbound network traffic
- Generate file creation activity
- Investigate multiple Sysmon events
- Correlate endpoint telemetry in Wazuh
- Build an attack timeline
- Map activity to MITRE ATT&CK

# Attack Scenario

A user executes PowerShell to access an external website.

The PowerShell process:

1. Starts execution
2. Establishes an outbound HTTPS connection
3. Creates a local file
4. Generates multiple Sysmon events
5. Sends telemetry to Wazuh

The SOC analyst investigates whether all observed events belong to the same activity.

# Detection Logic

The investigation correlates multiple endpoint events generated by a single PowerShell process.

By analyzing timestamps, Process GUIDs, command lines, and network activity, the analyst can determine whether the observed behavior belongs to a single execution chain.

The PowerShell process successfully generated Process Creation, Network Connection, and File Creation events.

Correlating these events enabled reconstruction of the complete execution timeline.
